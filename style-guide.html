We Eat Design System

A minimal, editorial design language built for a social recipe platform. Inspired by Nike.com — white backgrounds, sharp corners, bold typography, generous whitespace. Accent yellow is used sparingly as a single point of warmth.

Core Principles
Sharp, not rounded
All interactive elements use --btn-radius: 0px and --card-radius: 0px. Only chips use --radius-pill.
Uppercase with tracking
Labels, nav links, and section headers use 10px bold uppercase with letter-spacing-widest (0.12em).
One accent, used sparingly
--color-accent (#f2c438) appears in exactly one place per page — the gamification badge left-rule or a single interactive state. Never decoratively.
Pages
profile.html User profile — avatar, stats, gamification badge, recipe grid, tabs
recipe.html Recipe detail — hero, meta, ingredients, tools, directions
recipe-new.html Create / edit recipe form — dynamic ingredient groups, tools, direction sets
followers.html Followers / Following tabs — people list, search, follow buttons, mutual badge
recipes.html Popular recipes listing — filter chips, sort bar, recipe grid cards with rank badges
cookbook.html Cookbook detail — book cover, action bar with back button, recipe grid

Buttons

All buttons use .btn as a base. Sharp rectangular — border-radius: 0. Works on both <button> and <a> elements.

Variants
<button class="btn btn-primary">Publish</button>
<button class="btn btn-secondary">Cancel</button>
<button class="btn btn-ghost">View all →</button>
<button class="btn btn-ghost-muted">Skip</button>

btn-primary — Black fill, white text, uppercase widest tracking. Use for the primary action on a page.
btn-secondary — Black outline, inverts on hover. Use for secondary actions alongside a primary.
btn-ghost — No border, underlines on hover. Use for low-priority text links.
btn-ghost-muted — Same as ghost but starts muted. Use for truly tertiary actions.

Sizes
<button class="btn btn-primary btn-sm">Small</button>
<button class="btn btn-primary">Default</button>
<button class="btn btn-primary btn-lg">Large</button>
Icon Button
<button class="btn-icon" aria-label="Notifications">
  <svg ...>...</svg>
</button>

btn-icon — 40×40px tap target. Always include aria-label. Color starts at --color-text-muted, transitions to --color-text-primary on hover.

Ranks

Tier progression system across three rank tracks: Presence, Kitchen, and Palate. Each track has completed tiers (muted + checkmark), an active tier card with a progress bar, and locked tiers (muted + lock icon).

Tier States
Lurker 0 pts
Regular 100 pts
Contributor 250 – 499 pts
438 pts 62 pts to Influencer
Influencer 500 pts
Tastemaker 1000 pts

Completed — checkmark SVG, name + icon in --color-text-muted.
Active — bordered card (.tier-card), progress bar fill uses var(--color-accent) (page accent).
Locked — lock SVG, name in muted medium weight, icon in --color-border-strong.

Progress Bar
<div class="tier-card__bar-track">
  <div class="tier-card__bar-fill" style="width: 75%"></div>
</div>

Track is 4px tall, background: var(--color-bg-subtle). Fill uses background: var(--color-accent) — this is the page accent on ranks.html. Width set inline as a percentage of progress within the active tier.

Three Rank Tracks

Presence — social activity: posts, likes received, comments, followers, shares.
Kitchen — cooking content: recipes posted, saves received, cookbook entries.
Palate — food discovery: orders logged, places reviewed, cuisine variety.
